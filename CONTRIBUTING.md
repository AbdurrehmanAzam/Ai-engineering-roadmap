# Contributing

Thank you for your interest in this project! This is primarily a personal learning portfolio, but suggestions, corrections, and ideas are always welcome.

## How to Contribute

### 🐛 Reporting Issues
If you find a bug, broken link, or factual error:
1. Open a [GitHub Issue](https://github.com/AbdurrehmanAzam/Ai-engineering-roadmap/issues).
2. Describe the problem clearly and, where applicable, include the file path and line number.

### 💡 Suggesting Improvements
Have a resource, topic, or structural improvement to suggest?
1. Open an Issue with the label `suggestion`.
2. Briefly explain the improvement and why it would benefit the roadmap.

### 🔧 Submitting a Pull Request
1. Fork the repository and create a new branch from `main`:
   ```bash
   git checkout -b fix/your-description
   ```
2. Make your changes (keep them small and focused).
3. Commit with a clear message:
   ```bash
   git commit -m "fix: correct typo in phase-1 README"
   ```
4. Push your branch and open a Pull Request against `main`.
5. Describe what you changed and why in the PR description.

## Code Style
- Python files should follow [PEP 8](https://peps.python.org/pep-0008/).
- Markdown files should be readable and consistent with the existing style.
- Commit messages should use the conventional prefix style: `feat:`, `fix:`, `docs:`, `chore:`.

## Code of Conduct
Please read and follow the [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.
